# DONE CHECKLIST

Use this to decide whether the prototype is "done enough" for today.

A prototype does NOT need to be polished.
It DOES need to be honestly playable and judgeable.

---

## Core Playability

- [ ] The game launches correctly
- [ ] The player can start a run quickly
- [ ] Movement works reliably
- [ ] The main loop is playable
- [ ] The goal is clear
- [ ] The lose condition works
- [ ] A run can end cleanly
- [ ] The game can restart quickly

---

## Feedback / Readability

- [ ] Score or timer is visible
- [ ] The player can tell what is dangerous
- [ ] The player can tell what is collectible, if applicable
- [ ] Hit/death feedback is visible
- [ ] Basic UI is readable on web
- [ ] Visual clutter does not confuse play

---

## Minimum Juice

- [ ] At least one meaningful hit/death feedback effect exists
- [ ] At least one pickup or score feedback effect exists
- [ ] The game feels slightly responsive, not totally raw
- [ ] Restart flow feels fast enough to encourage retry

---

## Scope Discipline

- [ ] The build stays within the brief
- [ ] No major unnecessary systems were added
- [ ] The game still feels small and focused
- [ ] The first draft does not try to solve too many problems

---

## Technical Sanity

- [ ] No obvious broken references
- [ ] No major errors block play
- [ ] Important gameplay values are tuneable
- [ ] Shared/core systems were not changed recklessly
- [ ] The build remains suitable for web export

---

## Honest Evaluation

- [ ] I can judge the idea within 1–3 runs
- [ ] The game is understandable in about 5 seconds
- [ ] The game invites immediate replay
- [ ] There is at least one promising hook
- [ ] I know whether this is worth the rest of the day

---

## Shipping Decision

If most boxes above are checked, the prototype is ready for:
- polish pass
- balancing
- itch.io upload
- short capture for social media

If the core playability and replay urge are weak, stop expanding and move on to the next idea.
